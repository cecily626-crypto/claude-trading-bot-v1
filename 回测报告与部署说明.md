# crypto trading strategy 2.0-Short · 回测报告与部署说明

_2026-07-07 · 基于 crypto trading strategy 1.0（claude-trading-bot-v1）_

## 一、策略设计

在 1.0 的指标与状态机框架上比较了三种做空逻辑（代码见 `strategy_short.py`）：

- **S1 破位趋势做空**：跌破 N 根 K 线新低 + 确认下跌趋势（价格低于 EMA100 至少 3%、动量为负）时开空，ATR×2.5 移动止损离场。带三重反追杀过滤（单根暴跌不追、偏离破位线过远不追、RSI 超卖不追）。1.0 做多逻辑的镜像。
- **S2 泡沫反转做空**：meme 币单根暴涨 + RSI 超买 + 放量时做空博回落，快止盈/紧止损/限时持仓。
- **TS 趋势做空**：BTC/ETH 的 1.0 趋势策略只保留空头腿。

## 二、回测结果（LBank 4h · 2026-01-31 → 07-07 · 34 meme + BTC/ETH · 含费用滑点）

| 方案 | 笔数 | 胜率 | PF | 单笔均收 |
|---|---|---|---|---|
| **S1 最优（bo55+仅止损出场）** | 400 | 43.8% | **1.87** | +2.41% |

**结论：上线 S1(bo55/仅止损) + TS，放弃 S2。**

- 前半窗口 PF 2.78 / 后半 0.88 —— 纸账户阶段就是验证当前行情可行性，不达标不上真钱。

## 三、已部署的系统（仓库 claude-trading-bot-v1）

- `paper_bot_short.py`：独立做空纸账户 $2000（`paper_account_short.json`），每 5 分钟随 VPS 循环执行。单仓上限 meme 12% / BTC-ETH 40%；单日 -3% 熔断全平并停 24h；每笔开平仓 TG 推送（【2.0空】标签），送达保证（先落盘、失败重试 3 次、待发队列补发）。
- `daily_review_short.py`：每天 08:00（北京时间）随 daily-review Actions 推送昨日复盘。
- `weekly_review_short.py`：每周一随 weekly-review Actions 滚动重回测 + 与上周对比 + 纸账户周总结，恶化时给出优化方向。
- `backtest_short.py` + `backtest-short.yml`：策略选择回测，可随时在 Actions 手动重跑。
- `deploy/run_cycle.sh`：已加入 paper_bot_short 执行与状态推送。

## 四、你需要做的最后一步（激活 VPS）

VPS 的 systemd 执行的是 run_cycle.sh 的本地副本，SSH 登录 VPS（45.76.160.94）执行一条命令刷新：

```bash
cd /opt/trading-bot/repo && git pull --rebase --autostash && install -m 755 deploy/run_cycle.sh /opt/trading-bot/run_cycle.sh && systemctl start trading-bot.service && journalctl -u trading-bot.service -n 30 --no-pager
```

## 六、优化更新记录

**2026-07-14 · BAN 诱空复盘后加三道入场闸门**（起因：BAN 7-13 破位追空开在坑底 0.068 附近，随后深V反杀止损 ~-2%）：

- **opt#1 大盘环境闸门**（`paper_bot_short.REGIME_GATE`）：meme 破位空只在 BTC 4h 确认下跌（收盘 < EMA100 且 20 根动量为负）时才允许**新开**；数据缺失时保守不开。已持仓与 BTC/ETH 趋势空不受影响。
- **opt#2 回踩确认**（`strategy_short.breakdown_short(confirm=True, retest_window=6)`）：不再在创新低那根追空，改为破位后等一根"更高收盘＋更高低点、仍在下跌趋势"的回踩反抽才入场；若直接 V 型反转则不开。用真实 BAN 4h 数据验证：旧逻辑 7-12 20:00 开空，新逻辑该窗口**不开仓**。
- **opt#3 15m 微观择时**（`RSI15_MIN=35`）：新开空前查 15 分钟 RSI(14)，<35（超卖，易反弹）跳过；≥35 正常开（含 >70 超卖…注：>70 为超买，按用户设定允许开空）。15m 数据缺失时不拦。

⚠️ 这三条改变了 S1 的入场分布，**第二节的历史回测数字已不代表当前逻辑**。上线前需用 `backtest_short.py` 带 `confirm=True` 重跑，确认 PF/胜率/回撤不劣化再信任（15m 与大盘闸门是实盘 fetch，回测暂只覆盖 opt#2）。

**2026-07-15 · opt#1/#2 回测证伪，回滚现行**：在 `opt-backtest-ab` 分支用真实 LBank 4h 数据(34 meme + BTC/ETH，2026-01-31→07-15)A/B 重跑，opt#2 回踩确认把 S1 的 PF 从 **1.88 打到 0.95（总收益 +975%→-55%）**，opt#1 大盘闸门也劣化(H2 PF 0.69→0.59)。二者均**未采用**，`REGIME_GATE / RSI15_GATE = False`、`S1_KW.confirm=False`，即现行 `S1b bo55（仅止损出场）`。

**2026-07-15 · opt#4 滚动PF自适应仓位缩放（已写入纸账户 bot，✅ 采用）**

问题不在入场信号，而在"行情冷热"：S1 前半段(H1) PF 3.17、后半段(H2) PF 0.69——同一套信号在逆风段回吐。opt#4 不改信号，只调**每笔仓位大小**：用最近 `PF_N=18` 笔 meme(S1) 已平仓交易的盈利因子(PF)当"温度计"，冷则缩仓保命，热则满仓。

- **公式**：`系数 = 0.25 + (PF−0.9)/(1.4−0.9) × 0.75`，夹在 `[0.25, 1.0]`。PF≤0.9→0.25×(四分之一仓)，PF≥1.4→1.0×(满仓封顶，不加杠杆)，中间线性。
- **作用范围**：只缩 meme(S1) 新开仓的名义金额；入场触发/追踪止损/出场逻辑**不变**；BTC/ETH 趋势空不缩放。攒够 `PF_WARM=8` 笔平仓前一律满仓。全局窗口(不分币种)——测的是整个做空环境冷热。
- **参数**（`paper_bot_short.py` 顶部，一键开关 `PF_SIZING`）：`PF_N=18, PF_LO=0.9, PF_HI=1.4, PF_MULT_LO=0.25, PF_WARM=8`。取自稳健性扫描甜点区 15~20 的中点。
- **回测证据**（`opt-backtest-ab`，backtest #6）：稳健性扫描 24 格中 **N=15 与 N=20 两整行、四种阈值全部跑赢**满仓基线(ret/DD 6.44→8+)，是连成片非孤立尖峰。$2000 账户视角：满仓基线 **+35%／最大回撤 $503**，N20 缩放 **+40%／最大回撤 $315**——钱更多、回撤少约 37%。
- **局限（如实）**：只测了这一个 5.5 个月窗口；它是减损不是根治(H2 仍亏，只是亏更少)。纸账户实跑即进一步验证。
- **展示**：每笔开空 TG 消息在缩仓时带 `⚖️0.62×(PF1.15)`；每日复盘显示当前缩放系数与滚动PF。
